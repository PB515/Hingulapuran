# Hingulapuran — what we will build

The goal of this site is **digital conservation** of the Hinglaj Puran: the whole grantha (87 chapters) turned into a faithful, readable, beautiful archive that a stranger can understand and that a community can be proud of. Working translations stay marked as drafts until a pandit reviews them.

This is the running build list. It is grouped by theme, with a rough size on each item (S = a sitting, M = a day, L = several days). The order at the bottom is a suggestion, not a rule. Tick items as they ship.

---

## What already exists (done)
- The premium site shell: home with the Fall of Sati cinematic scroll, plus Books, Blog, Gallery, Temples, Contact, and a working Admin (auth, posts, inbox).
- The `/grantha` archive: a reader-facing front door, ten curated docs rendered clean (production notes stripped), plain-language intros, and the full translation now read **one chapter per page** with previous/next.
- The craft elements library it draws on (cinematic scroll, gateway, horizontal scroll, and more).

---

## A. Make the grantha deeper (the north star)
- [ ] **A1. Finish the translation, ch 68 to 87** *(L, content)* — the per-chapter reader already grows on its own as each chapter lands in the source. This is the core conservation work.
- [ ] **A2. The original alongside the translation** *(M)* — show the Gujarati (or the scanned page) next to the English on each chapter page, so a reader can check the source. The heart of "faithful."
- [ ] **A3. Downloadable original** *(S)* — a clear "download the original 1915 book" action, so the artifact itself is carried forward, not only our reading of it.
- [ ] **A4. Glossary cross-links** *(M)* — names in the text (Hingula, Parashurama, Mahishasura) link to their entry in "Who's Who," so a reader never gets lost.
- [ ] **A5. Search across the grantha** *(M)* — one search box over every chapter, name, and place.

## B. Make it cinematic (the craft layer)
- [ ] **B1. The Sacred Arc as a visual timeline** *(L)* — turn "The Story, Start to End" (the seven movements) into a scroll experience, not a wall of text. This is the showpiece.
- [ ] **B2. Per-chapter art** *(L, content + build)* — one image per chapter from the scene prompts, shown at the head of each translation page. Turns the archive into something people want to look at.
- [ ] **B3. The Hingul and Parashurama reels** *(M each)* — the two big episodes as their own cinematic scroll sections, like the Fall of Sati already on the home page.
- [ ] **B4. The sacred-places map** *(M)* — wire the tirtha gazetteer into the existing temple map so the geography of the story is something you can explore.
- [ ] **B5. The family trees, drawn** *(S)* — render the styled genealogy artwork on its page instead of a plain list.

## C. Reader trust and polish
- [ ] **C1. Review status, visible** *(M)* — a clear, honest marker on each chapter ("working draft" vs "reviewed by a pandit"), and a simple way to track which chapters are approved. This protects the project's credibility.
- [ ] **C2. Gujarati and Devanagari typography pass** *(S)* — make sure every Indic line is set in the right font at the right size, nothing broken.
- [ ] **C3. Accessibility and reading comfort** *(S)* — font-size control, good contrast, calm default dark reading surface.

## D. Ship it
- [ ] **D1. Deploy to a real URL** *(S)* — get it live so the client can see and share it.
- [ ] **D2. Share cards and SEO** *(S)* — when a chapter or the grantha is shared, it looks like a real publication.
- [ ] **D3. Connect Blog and Gallery to the mission** *(S)* — the blog tells the conservation story; the gallery holds the scans and the art.

---

## Suggested order
1. **B4 sacred-places map** and **B5 family trees** — quick, high-impact, the data already exists.
2. **A2 original alongside translation** + **A3 download** — the credibility core of "conservation."
3. **B1 the Sacred Arc timeline** — the showpiece moment.
4. **A4 glossary links** + **A5 search** — make the archive navigable.
5. **B2 per-chapter art** + **B3 the two reels** — as the art and translations come in.
6. **C and D** — review-status, polish, and ship.

> Pick the next item and we build it one layer at a time (static, then pacing, then motion, then polish), verifying each step.
